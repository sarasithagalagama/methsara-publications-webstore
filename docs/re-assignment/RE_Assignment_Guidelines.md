# Requirements Engineering - Assignment 1 & 2 Guidelines

## Objective

Assignment 1 and 2 are closely related to each other. In these two assignments, you will analyse the requirements and write a requirements specification for a software development project of your own choice from the problem statements provided. Assignment 1 expects you to model the problem using a mixture of modelling techniques.

## Doing the Assignment 1

Your group may use requirement elicitation techniques for information-gathering (e.g. interviews) to collect any additional information you need to refine the requirements provided in the problem statement.

### Tasks

1. **Identify the system stakeholders**

2. **Identify all the functional requirements (FR) and non-functional requirements (NFR), constraints**

3. **Develop Strategic Dependency (SD) diagram** to model the strategic dependencies between the key stakeholders/actors and capture the organisational context

4. **Develop a high-level Business Model** using Business Model Canvas

5. **Develop a UML Use Case Diagram** to model the high-level behaviour of system and provide 5 or 6 scenarios

6. **Implement the Agile/Scrum planning and estimation** to your project:
   - Develop a Product backlog table: user stories, story points, and sprint number
   - Assign a point value to each task based on its complexity
   - Assign a sprint/iteration number to each task (For example, if you have 4 weeks to submit the assignment, you can consider 4 sprints to complete it with each sprint lasting 7 days)
   - Design a burn-down/burn-up chart and provide an explanation (Note: you can use Excel to design the table and burn-up chart)
   - Develop a requirements traceability matrix (RTM)
